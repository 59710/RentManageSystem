## 一、项目概述

### 1.1 基本信息

| 属性 | 内容 |
|:---|:---|
| **项目名称** | 青年品质租房管理系统（Youth Premium Rental System） |
| **项目类型** | Web全栈开发 / 前后端分离架构 |
| **项目周期** | 2026年5月 — 至今（持续迭代中） |
| **项目角色** | 项目负责人 & 全栈开发工程师 |
| **项目定位** | 面向18-35岁青年群体的在线租房管理服务平台 |

### 1.2 项目背景

随着高校毕业生数量逐年攀升，青年群体的租房需求日益旺盛。传统租房市场长期存在**信息不对称、房源真实性难辨、交易流程繁琐、合同管理混乱**等痛点。本项目致力于打造一套前后端分离架构的品质租房管理平台，整合房源发布审核、在线签约支付、物业报修、数据统计等核心业务能力。

### 1.3 核心价值

- **信息透明**：所有房源经人工审核，杜绝虚假信息
- **安全保障**：实名认证 + BCrypt加密 + JWT Token认证 + HTTPS全链路
- **便捷高效**：在线浏览→预约看房→申请租房→电子签约→在线支付全流程线上化
- **品质服务**：专业客服响应 + 快速报修处理 + 数据驱动运营决策

## 四、核心亮点与技术实践

### 4.1 安全设计

| 安全措施 | 实现方案 | 防护目标 |
|:---|:---|:---|
| **密码安全** | BCrypt（强度因子=12），废弃MD5 | 防彩虹表/暴力破解 |
| **身份认证** | JWT Bearer Token + Spring Security过滤器链 | 无状态认证 |
| **SQL注入防护** | MyBatis `#{}` 参数化查询，禁用字符串拼接 | OWASP Top 1 |
| **XSS防护** | 输入过滤（HTML实体转义）+ 输出编码 | 跨站脚本攻击 |
| **敏感数据保护** | 手机号脱敏显示（138****8000），身份证AES加密存储 | 数据泄露防护 |
| **接口权限** | RBAC + 方法级权限控制 | 垂直/水平越权 |
| **操作审计** | 全量操作日志记录（模块/动作/IP/耗时/结果） | 安全追溯 |
| **登录防护** | 5次失败锁定30分钟 | 暴力破解 |
| **CSRF防护** | SameSite Cookie + Token双重校验 | 跨站请求伪造 |

### 4.2 架构设计亮点

- **统一响应封装**：`Result<T>` 泛型类封装所有API响应，`{code, message, data}` 三字段结构，前端统一处理
- **全局异常处理**：`@RestControllerAdvice` 捕获所有异常，业务异常、参数校验异常、系统异常分级处理
- **DTO分层**：Request/Response/VO分离，避免实体直接暴露，敏感字段脱敏返回
- **分页标准化**：`PageQuery` 统一分页参数，`PageResult` 包含 `total/page/size/totalPages/hasPrevious/hasNext`
- **JWT过滤器**：`OncePerRequestFilter` 保证单次请求只过滤一次，跳过OPTIONS预检请求
- **自动填充**：MyBatis-Plus `MetaObjectHandler` 自动填充 `create_time` / `update_time`
- **枚举字典**：前后端统一的20组枚举值（角色/状态/类型/渠道等）

### 4.3 前端工程化实践

- **TypeScript严格模式**：类型安全，编译期错误检查
- **Axios拦截器**：请求拦截自动附加Token，响应拦截统一错误处理（401跳转登录）
- **Pinia状态管理**：用户登录状态持久化（localStorage），角色计算属性（isTenant/isLandlord/isAdmin）
- **路径别名**：`@/` 映射 `src/`，减少相对路径引用
- **Vite代理**：开发环境API代理到后端，解决跨域问题
- **Element Plus**：企业级UI组件，统一的交互体验
